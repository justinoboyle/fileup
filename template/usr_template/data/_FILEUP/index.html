fileup documentation

Variables

$INSTALL_ROOT: fileup's install root
$FILEUP_USERNAME: a user's name
$FILEUP_TOKEN: a user's token
$FILEUP_HOST: the root web url

Create a user

Take a look at $INSTALL_ROOT/usr/config/users.json.

Example:

{
    "$FILEUP_USERNAME": {
        "token": "Leave this blank and visit the user's home directory ($FILEUP_HOST/$FILEUP_USERNAME); the key will be generated on the first page visit."
    }
}

Setting root hosts

Users can have their own root hosts if they want (multiple users & multiple domains)
Take a look at $INSTALL_ROOT/usr/config/users.json.

Example:

{
    "$FILEUP_USERNAME": {
        "token": "Leave this blank and visit the user's home directory ($FILEUP_HOST/$FILEUP_USERNAME); the key will be generated on the first page visit."
        "rootOn": [
                "https://i.theUsersSite.com"
        ]

    }
}

Pushing Something

Users can easily push things using curl or a web form.

curl -F 'username=$FILEUP_USERNAME' -F 'token=$FILEUP_TOKEN -F 'file=@filename.ext' $FILEUP_HOST

Alternatively, you can make an alias, and save your key and host as an environment variable. (sh/bash/zsh example)

function _fileup() { curl -F "username=$FILEUP_USERNAME" -F "token=$FILEUP_TOKEN" -F "file=@$1" $FILEUP_HOST }; alias fileup=_fileup

Homepages

Users can easily create homepages by pushing an index.html file. Files with the names index.html, style.css, _404.html, and script.js will always be named properly and non-scrambled.
